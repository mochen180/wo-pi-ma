# LLM 聊天记录总结工具

## 功能介绍

本工具基于practice02中的代码，增加了聊天记录总结功能，当满足以下条件之一时，会自动触发聊天记录总结：

1. 聊天历史记录超过5轮（即10条消息，包括用户和助手的消息）
2. 聊天上下文长度超过3000个字符

## 总结逻辑

当触发总结时，工具会：
1. 对前70%左右的聊天内容进行压缩总结
2. 保留最后30%左右的聊天内容原文
3. 将总结结果作为一条特殊的消息添加到聊天历史中
4. 用总结后的内容替换原来的聊天历史，从而减少上下文长度

## 使用方法

### 新代码1：聊天记录总结工具

1. **配置环境**：
   - 默认使用本地部署的AI服务（如Ollama）
   - 默认配置：
     ```
     BASE_URL=http://localhost:11434/api/chat
     MODEL=llama3
     API_KEY=
     ```
   - 如果需要使用其他本地部署的AI服务，可以在项目根目录下创建 `.env` 文件进行配置

2. 运行工具：
   ```bash
   python llm_client_with_summary.py
   ```

### 新代码2：关键信息提取与聊天历史查询工具

1. **配置环境**：与新代码1相同

2. 运行工具：
   ```bash
   python llm_client_with_keyinfo.py
   ```

3. **功能说明**：
   - 每5次聊天自动提取关键信息（按照5W规则）
   - 关键信息保存到 `D:\chat-log\log.txt`
   - 使用 `/search` 开头或表达 "查找聊天历史" 的意思可以搜索聊天历史
   - 当提到 "文档仓库"、"文件仓库"、"仓库" 时自动触发 AnythingLLM 查询
   - 支持所有原有的文件操作和互联网搜索工具

## 新功能：AnythingLLM 查询

### 配置要求
1. 在 `.env` 文件中配置以下变量：
   ```
   ANYTHINGLLM_API_KEY="your_anythingllm_api_key"
   ANYTHINGLLM_WORKSPACE_SLUG="your_workspace_slug"
   ```

### 使用方法
1. 当你需要查询文档仓库中的内容时，可以使用以下方式：
   - 直接提到 "文档仓库"、"文件仓库" 或 "仓库"
   - 明确要求查询文档内容

### 示例
- **输入**：在文档仓库中查找关于机器学习的内容
- **输出**：系统会自动调用 AnythingLLM API 并返回查询结果

3. 输入你的问题或文件操作请求，按Enter发送。按Ctrl+C退出。

## 工具列表

本工具支持以下操作：

- `list_files`: 列出某个目录下的所有文件和目录
- `rename_file`: 修改某个目录下某个文件的名字
- `delete_file`: 删除某个目录下的某个文件
- `create_file`: 在某个目录下新建1个文件，并且写入内容
- `read_file`: 读取某个目录下的某个文件的内容
- `search_internet`: 根据用户对话的内容搜索互联网

## 注意事项

1. 本工具需要有效的OpenAI API密钥才能正常工作
2. 总结功能会调用LLM API，因此会产生额外的token消耗
3. 总结后的聊天历史会保留最近的对话内容，以确保上下文的连贯性
